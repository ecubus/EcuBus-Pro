# 功能请求流程

## 步骤

1. 在 GitHub 仓库中创建功能 [issue](https://github.com/ecubus/EcuBus-Pro/issues/new/choose)。
2. 清楚描述您想要添加的功能，包括它的目的以及它将如何使用户受益。
3. 使用 "增强" 标记问题以适当分类.

为了确保我们优先考虑有利于社区的功能，新功能请求至少需要 5 个GitHub 反应(👍) 才能考虑我们的积压。

## 如何进行投票

添加 👍 响应功能请求问题。

## 进程

1. 创建功能请求问题
2. 社区成员通过 👍
3. 有5+个响应，将审查该功能以了解待办事项
4. 没有足够反应的功能可能被关闭
